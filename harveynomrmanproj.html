Harvey Norman E-commerce Platform

Full-stack e-commerce platform, integrating sustainability features aligned with Singapore's Green Plan 2030 while demonstrating comprehensive app development skills.

Project Scope & Simulation Context

Simulated Scenario

This project simulated a Harvey Norman website, addressing their need to establish an e-commerce presence while aligning with Singapore's Green Plan 2030 sustainability objectives.

What were the main goals, challenges & solutions?

Goals:

  • Full-Stack Demonstration: Showcase comprehensive web development skills using Python Flask, Bootstrap, and modern technologies
  • Functional E-commerce Prototype: Create working online store with inventory management and delivery tracking features
  • Sustainability Integration: Incorporate environmental concepts aligned with Singapore's Green Plan 2030
  • Software Engineering Principles: Apply scalability, maintainability, and user experience best practices
  • Timeline Management: Complete all academic requirements within 12-week semester constraints
  • Professional Documentation: Deliver comprehensive project documentation and presentation materials

Challenges & Solutions:

  • Learning Curve: Solved by dedicated study time, Flask tutorials, and practice applications
  • Feature Scope: Addressed through MVP implementation with phased enhancements
  • Data Management: Achieved using Python Shelve module for lightweight persistence
  • Sustainability Integration: Accomplished by studying Green Plan 2030 and adding carbon footprint features
  • Presentation Preparation: Completed with dedicated documentation weeks and multiple practice sessions

My Role and Technical Solution

Academic Work Process & Project Management

How did you plan, manage and complete the project on time?

Weeks 1-2: Ideation & Proposal Development
• Brainstormed multiple e-commerce website concepts as a team
• Developed comprehensive project proposal outlining scope and objectives
• Defined how sustainability would be integrated across different site sections
Weeks 3-4: Role Assignment & Individual Planning
• Divided the website into logical components based on team member strengths
• Each member selected specific pages/features to develop independently
• Created detailed specifications for integration points between components
• Established shared design standards (color scheme, typography, layout)
• Set up shared repository and development environment
Weeks 5-8: Independent Development Phase
• Each team member coded their assigned pages/features independently
• Regular weekly meetings to share progress and troubleshoot issues
• Maintained consistent communication via WhatsApp group for quick questions
• Shared code snippets and design elements to ensure visual consistency
Weeks 9-10: Integration & Testing
• Combined all individual components into a unified website
• Resolved integration challenges and interface mismatches
• Ensured all navigation links worked correctly between sections
• Tested sustainability features across different user scenarios
Weeks 11-12: Finalization & Presentation
• Refined integrated website based on testing feedback
• Prepared final documentation and project report
• Created presentation materials highlighting individual contributions
• Submitted final project and presented to class

Our Collaborative Workflow

Our Team Collaboration Strategy:

  • Weekly Team Meetings: Regular sync-ups to align progress and address integration challenges
  • Independent Development: Each team member responsible for specific website components
  • Integration Sessions: Dedicated meetings to combine individual work into a cohesive website
  • Code Review: Peer review of each other's code for quality assurance and consistency
  • Iterative Refinement: Multiple rounds of feedback and continuous improvement
  • Version Control: Using Git for collaborative development and change management
